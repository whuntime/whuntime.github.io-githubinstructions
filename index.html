Project Assumptions and scope statement

This instruction assumes that the "new programmer" knows their trade-craft well enough to instantiate their concept with html, css, & JavaScript. Further, it assumes
that they don't have a GitHub account and that they need step by step instruction on how to accomplish their goals.


Note: You will need a basic html page along with any referenced resources images, music, videos, etc. This part is dependent on site requirements.


GitHub account creation


Navigate to www.github.com, click Sign up in the upper right corner of the page



On the resulting page type in your e-mail address, then click continue



Then enter a password and click continue



Once signed and on your home page select the down arrow next to the + icon on the upper right of the screen or your user icon then select new repository. After you have
created the repository you will access your existing repositories from the user drop-down and selecting Your repositories



On the Create a new repository page you will name your repository. It should be a name that logically matches the purpose of the site you are creating. Note that Public is
the default setting for repositories so this will not need to be changed for this project. Further, no optional selections will be made. Click Create repository



Navigate to your repositories, click on the new repository you created



Select Add File then Upload files



From your computer navigate to the files you have prepared for project upload and drag those files to the window or select the choose your files link on the page navigate to
the files and select them for upload



Once files are posted and optionally commented you can then navigate to the page. In this case the page would be yourname.yourproject.io
